NetMarket

Le but était de créer un site e-Commerce en utilisant le framework SpringBoot pour le back-end, au choix pour le front-end.

Le site comprend une page de catalogue, une page de détail, un panier, une page de paiement, une page de connexion ou encore une page d'inscription
Il y avait donc un back-office avec une gestion des utilisateurs et leurs droits, une gestion des produits, des catégories, des commandes...
Après une analyse et l'établissement du MCD, nous nous sommes surtout concentré sur le back-end en créant les modèles et services.

SoluRisk

Le but était de créer une application web from scratch pour un vrai client.

A l'origine, le client possédait un site web en WordPress, mais il voulait une application web plus personnalisée.
Après une analyse des besoins, nous avons utilisé le framework Laravel pour le back-end, et nous avons dû respecter un cahier des charges validé avec le client.
Nous avons ensuite adopté une méthode agile simple pour la conception afin de s'adapter aux besoins qui allaient évoluer.
Ainsi, nous avons développé un front-office pour les clients de Solurisk et un back-office pour les administrateurs.
C'était le gros projet de l'année et c'était la première fois que je travaillais pour un vrai client, et cela m'a permis de découvrir les contraintes du monde professionnel.

Logiciel de tableau de bord

L'objectif était de créer un logiciel en Windev permettant à chaque service d'accéder à plusieurs outils complémentaires à notre ERP, notamment des affichages spécifiques.

Un autre objectif était de permettre à chaque service de suivre ses indicateurs de performance.
J'ai aussi fait d'autre outils permettant de mettre à jour les signatures de mails ou permettant de lancer d'autres opérations diverses.
Ce logiciel devait communiquer avec la base de données de l'ERP de l'entreprise, contenant plus de 100 tables relationnelles.
J'ai donc dû apprendre à optimiser les requêtes SQL et à comprendre l'architecture de la base de données pour ne pas la surcharger.
